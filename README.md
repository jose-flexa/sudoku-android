# SDD Project

This repository contains a Software Design Document (SDD) structure.

## Quick Start

1. Fill in project information in `docs/sdd/00-project-charter.md`.
2. Capture requirements in `docs/sdd/01-requirements.md`.
3. Define architecture in `docs/sdd/02-system-architecture.md`.
4. Complete detailed modules in `docs/sdd/03-detailed-design.md`.
5. Link requirements to design and tests in `docs/sdd/09-traceability-matrix.md`.

## Structure

- `docs/sdd/`: Main design documentation.
- `docs/adr/`: Architecture Decision Records (optional but recommended).
- `docs/diagrams/`: Diagram sources and exports.

## Suggested Workflow

- Update requirements first.
- Keep ADRs small and incremental as decisions are made.
- Update traceability whenever requirements or tests change.
